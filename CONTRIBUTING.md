# Contributing to Claude Interface

Merci de votre intérêt pour contribuer à Claude Interface ! 🎉

## Comment contribuer

### Signaler un bug

1. Vérifiez que le bug n'a pas déjà été signalé dans les [Issues](https://github.com/julien/claude-interface/issues)
2. Créez une nouvelle issue avec le template "Bug Report"
3. Incluez des détails précis :
   - Version de l'application
   - Navigateur et version
   - Steps to reproduce
   - Comportement attendu vs réel
   - Screenshots si possible

### Proposer une feature

1. Vérifiez la roadmap dans le README
2. Créez une issue avec le template "Feature Request"
3. Décrivez clairement la feature et son utilité

### Soumettre du code

1. **Fork** le repository
2. **Clone** votre fork : `git clone https://github.com/votre-username/claude-interface.git`
3. **Créez une branche** : `git checkout -b feature/ma-feature`
4. **Commitez** vos changements : `git commit -m 'Add: ma super feature'`
5. **Push** vers la branche : `git push origin feature/ma-feature`
6. **Ouvrez une Pull Request**

### Standards de code

- Utilisez TypeScript
- Suivez les conventions ESLint configurées
- Écrivez des composants fonctionnels avec hooks
- Commentez le code complexe
- Nommage en anglais pour le code, français pour l'UI

### Style de commit

Utilisez des messages clairs :
- `Add: nouvelle fonctionnalité`
- `Fix: correction de bug`
- `Update: mise à jour de feature existante`
- `Refactor: refactoring sans changement de comportement`
- `Docs: mise à jour documentation`

### Tests

Avant de soumettre :
```bash
npm run lint    # Vérifier le linting
npm run build   # Vérifier que le build passe
npm run dev     # Tester en local
```

## Code de conduite

- Soyez respectueux et professionnel
- Accueillez les nouveaux contributeurs
- Acceptez les critiques constructives
- Concentrez-vous sur ce qui est meilleur pour la communauté

Merci de contribuer ! 🚀
